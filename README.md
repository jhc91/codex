Best Practices
思考这个问题的一个有用方法是：
  从正确的任务背景开始，①使用 AGENTS.md 获取持久的指导，②配置 Codex 以匹配您的工作流程，③将外部系统与 MCP 连接，④将重复性工作转化为技能，⑤并实现稳定工作流程的自动化。


### 提示词中包含
#### 目标：你想要改变什么或建立什么
#### 上下文：哪些文件、文件夹、文旦蛋糕或者错误与此任务有关
#### 限制条件：Codex 应该遵循哪些标准、架构、安全要求
#### 完成条件：任务完成前应该满足什么条件，例如：。。。。



### 首先要为困难的任务做好计划
#### 使用计划模式
#### 使用codex对你的想法进行重构
#### 使用plan.md模版



### 使用agnets.md使指南可重复使用
一份好的 AGENTS.md 文件应包含以下内容：
仓库布局和重要目录
如何运行该项目
构建、测试和代码检查命令
工程惯例和公关预期
约束和禁止规则
完成工作的含义以及如何验证工作



### 将重复性工作转化为技能
一旦工作流程变得可重复，就不要再依赖冗长的提示或反复的沟通。使用技能（Skill） 将指令、上下文和支持逻辑打包到 SKILL.md 文件中，Codex 应始终如一地应用这些逻辑。
